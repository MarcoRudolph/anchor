## Trust Boundaries

| Hermes/edge fn (service-role) → Postgres | RLS bypassed; handler authz required |
| At-rest token storage | Google refresh tokens encrypted at rest |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-06 | Information Disclosure | RLS-missing user-owned table | mitigate | 0004 enables RLS with `auth.uid()` policies on EVERY user-owned table from the first schema; the verify grep + checkpoint psql check confirm coverage. |
| T-00-07 | Information Disclosure | Cleartext Google refresh token | mitigate | `google_connection` stores only a pgcrypto-encrypted refresh-token column; NO access-token column; cleartext never persisted (DEC-0008). |
| T-00-08 | Tampering | Cross-source ambiguity in memory index | mitigate | `memory_index_item` single-source CHECK enforces exactly-one-FK + matching source_kind (DEC-0004), preventing a memory item claiming two evidence sources. |
| T-00-09 | Elevation of Privilege | Service-role client reaching browser | mitigate | `src/db/client.ts` keeps the service-role client server-only and non-browser-importable; webapp uses anon + user session through RLS. |
| T-00-10 | Repudiation | Account actions without audit trail | mitigate | `audit_event` table frozen now (anchor_user_id nullable for 12-month anonymization), ready for recovery/deletion/pairing audit writes in later waves. |
</threat_model>
